# Elegant Home Automation Project Instructions

## Project Identity

This repository powers Elegant Home Automation:

https://www.eleganthomeautomation.com/

Elegant Home Automation is a premium smart home design and installation company serving Orange County, California. The site should position the business as a thoughtful, high-end partner for homeowners who want smart technology to feel simple, beautiful, reliable, and carefully integrated into the home.

## Brand Direction

All future work should reflect the Elegant Home Automation brand:

- Premium
- Luxury-oriented
- Design-conscious
- Architectural
- Lifestyle-focused
- White-glove service

The brand should never feel generic, contractor-like, overly technical, gadget-heavy, or mass-market.

## Design Language for SEO Landing Pages

All future SEO pages must match the existing Elegant Home Automation design language.

Preserve:

- Existing global header
- Existing footer
- Existing homepage hero
- Existing brand palette and typography patterns

Use:

- Dark charcoal backgrounds
- Warm gold accents
- Luxury residential imagery
- Elegant typography
- Generous spacing
- Calm, premium visual hierarchy
- Clean cards, refined borders, subtle shadows, and polished image treatments

Avoid:

- Generic contractor language
- Overly technical copy
- Fake testimonials
- Unverified certifications
- Claims of completed projects unless explicitly provided
- Dense text blocks
- Bright or off-brand colors

## Standard SEO Page Architecture

SEO landing pages should generally follow this structure:

1. Hero section
2. Trust section
3. Main content sections
4. Local SEO section
5. FAQ section
6. CTA section
7. Footer

Use semantic HTML. Each page should feel like a polished landing page, not a plain blog article.

## Internal Linking Rules

Every SEO page must include:

- A crawlable link to the homepage
- Links to relevant service pages
- Links to related SEO pages
- Footer links where appropriate

The homepage must link back to every SEO page. Add these links in a compact, premium way that does not clutter the homepage.

All internal links must be real anchor links using the appropriate Next.js `Link` component.

## Navigation Requirements

All SEO pages must appear under:

Blog -> SEO Pages

The Blog menu must remain visible in desktop navigation.

Mobile navigation must also expose SEO pages so visitors can reach them without desktop hover behavior.

Do not remove or visually redesign the global navigation unless explicitly requested.

## SEO Requirements

Every SEO landing page must include:

- Unique SEO title
- Unique meta description
- One H1 only
- FAQ schema
- LocalBusiness or Service schema when appropriate
- Sitemap inclusion
- Crawlable internal links
- Descriptive image alt text
- Natural local SEO language for Orange County and relevant cities

Avoid duplicate city-page copy. Local pages should include specific local context where possible.

## Responsive Requirements

All pages must work cleanly across:

- Desktop
- Tablet
- Mobile

Requirements:

- No horizontal scrolling
- Hero text remains readable on mobile
- Cards stack cleanly
- FAQ sections remain usable
- Navigation remains accessible
- Images use responsive sizing and object-fit behavior
- Buttons and text must not overlap

## Quality Checks

Before finishing future SEO landing page work, verify:

- The page builds successfully
- There are no TypeScript errors
- The page URL exists
- Homepage backlink exists
- Blog -> SEO Pages entry exists
- Mobile navigation exposes the page
- Relevant service links are present
- Related SEO links are present
- FAQ schema is included
- Sitemap includes the page
- Mobile layout is clean and usable

If local browser verification is not possible, state that clearly and report the build result.
